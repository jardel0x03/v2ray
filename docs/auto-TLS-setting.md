# V2Ray 脚本 no-auto-tls 参数帮助说明

2023-05-13 12:24

如果你想要手动配置 TLS，请使用 no-auto-tls 参数来添加配置，例如你想要用 NGINX 实现 TLS

## 前言

考虑到有些人不方便使用 Caddy 实现自动 TLS，比如已使用 NGINX 跑了自己的博客，又或者觉得 Caddy 性能不够好之类等…

V2Ray 脚本自带了一个 no-auto-tls 参数可以让你非常方便的让你来手动配置 TLS

## 备注

此处说的手动配置 TLS，指的是使用 WEB 程序转发 ws/h2/grpc 流量到 V2Ray

如图片上的 VMess-WS-TLS 协议可以参考使用 此 NGINX 配置模板

其他配置模板？都手动了，你手动一下吧

## 提醒

no-auto-tls INFO 这个配置信息可以使用 `v2ray info tls` 指令查看配置的时候会自动显示

或者你也可以使用 `v2ray debug tls` 指令来查看，你可以找到 port 和 path 相关的信息

但是，如果是 gRPC 的路径，你得理解一下是怎么处理的才行

## 缺点

都说了嘛，手动

所以如果你改了路径之类，又得自己来手动配置一次。

## 懂的都懂

不要好奇为什么显示配置的时候端口会是 443，而 no-auto-tls INFO 下面又是不同的端口。
